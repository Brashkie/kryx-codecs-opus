## ⚠️ Status: SKELETON (v0.1.0-alpha.0)

The public API surface is finalized but `encode()` / `decode()` are stubs
that throw `CodecError('unsupported')`.

The libopus 1.5.2 sources are vendored (`vendor/libopus/`) — the C source code
is in place. What's pending is the **Zig build script** + **Rust FFI** to
turn those sources into a usable encoder/decoder.

The API contract below is stable and will not change between alpha → stable.

See [docs/IMPLEMENTATION.md](docs/IMPLEMENTATION.md) for the milestone roadmap.

## Install

```bash
npm install @kryxjs/codecs-opus
```

> The right native binary for your platform is installed automatically via
> `optionalDependencies`. Supported: Windows x64/arm64, macOS x64/arm64,
> Linux x64 (gnu/musl), Linux arm64 (gnu).

## Usage

### Auto-registration (default)

```ts
import '@kryxjs/codecs-opus'             // side-effect: registers 'opus'
import { createDecoder } from '@kryxjs/codecs'

const decoder = createDecoder('opus', { sampleRate: 48000, channels: 2 })
```

### Explicit registration

```ts
import { registerOpus } from '@kryxjs/codecs-opus/register'
registerOpus()
```

### Direct class usage

```ts
import { OpusDecoder, OpusEncoder, OpusApplication } from '@kryxjs/codecs-opus'

const enc = new OpusEncoder({
  sampleRate: 48000,
  channels: 2,
  application: OpusApplication.Audio,
  bitrate: 128_000,
})

const packet = await enc.encode(frame)
```

## Configuration

```ts
interface OpusConfig {
  sampleRate?: 8000 | 12000 | 16000 | 24000 | 48000  // default 48000
  channels?: 1 | 2                                    // default 2
  application?: 'voip' | 'audio' | 'lowdelay'         // default 'audio'
  bitrate?: number                                    // default 64000
}
```

## Architecture

```
@kryxjs/codecs-opus (npm package)
    ↓ TypeScript façade (src/)
    ↓
@kryxjs/codecs-opus.<platform>.node (per-platform binary)
    ↓ napi-rs bindings (crates/opus-node/)
    ↓
opus-core (Rust core, crates/opus-core/)
    ↓ extern "C" FFI (generated by bindgen at build time)
    ↓
zig/src/opus_shim.zig (Zig thin wrapper, exposes clean C ABI)
    ↓
vendor/libopus/ (libopus 1.5.2 C sources, BSD-3-Clause)
```

## Why Zig?

[Zig](https://ziglang.org) handles the C build of libopus with a single
command (no need for autoconf/configure on all platforms). It also gives
us free cross-compilation, consistent SIMD detection (SSE/AVX/NEON), and
deterministic builds.

## Development

### Prerequisites

- Rust ≥1.80 (`rustup install stable`)
- Zig 0.13.x ([download](https://ziglang.org/download/))
- Node.js ≥18
- bindgen (`cargo install bindgen-cli`)

### Setup

```bash
git clone https://github.com/Brashkie/kryx-codecs-opus.git
cd kryx-codecs-opus
npm install
npm run build:debug
npm test
```

### Repository layout

```
kryx-codecs-opus/
├── src/                     TypeScript layer (OpusEncoder, OpusDecoder, types)
├── crates/
│   ├── opus-core/           Rust core (skeleton)
│   └── opus-node/           napi-rs bindings (skeleton)
├── zig/                     Zig build script + C ABI shim (M2 pending)
├── vendor/libopus/          libopus 1.5.2 vendored sources (BSD-3-Clause)
├── __tests__/               Vitest tests (API contract today, real tests later)
├── docs/
│   └── IMPLEMENTATION.md    The 8-milestone plan
├── scripts/                 Build helpers
└── .github/workflows/       CI / Release
```
